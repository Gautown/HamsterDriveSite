### 系统要求

- 操作系统：Windows 10 或更高版本
- Rust 工具链：Rust 1.93.0+
- 构建工具：MinGW-w64 或 Visual Studio Build Tools
</Prose>

<div class="space-y-4">
<Prose>
### 从源码构建

克隆仓库并构建发布版本：
</Prose>

```bash
# 克隆仓库
git clone https://github.com/Gautown/HamsterDrivers.git
cd HamsterDrivers

# 构建发布版本
cargo build --release

# 运行程序
cargo run
```

<Prose>
或者直接运行构建后的可执行文件：

```bash
./target/release/hamster-driver-manager.exe
```

#### 使用构建脚本

项目提供了便捷的构建脚本：

```bash
# 使用 PowerShell 脚本构建
.\build.ps1

# 使用批处理脚本构建
package.bat
```

#### 预编译版本

可以从 [Releases 页面](https://github.com/Gautown/HamsterDrivers/releases) 下载预编译的可执行文件。

</Prose>
</div>

### 环境配置

#### 安装 Rust 工具链

```bash
# 安装 Rust 工具链
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh

# 安装 Windows 目标
rustup target add x86_64-pc-windows-gnu

# 安装构建依赖（Windows）
# 需要安装 MinGW-w64 或 Visual Studio Build Tools
```

### 开发命令

```bash
# 开发模式运行
cargo run

# 调试构建
cargo build

# 发布构建
cargo build --release

# 运行测试
cargo test

# 代码格式化
cargo fmt

# 代码检查
cargo clippy
```

</Prose>

### Manual setup

#### 1. Add the cn utility

Create `src/utils/cn.ts`:

```typescript
export function cn(...classes: (string | undefined | null | false)[]): string {
  return classes.filter(Boolean).join(' ');
}
```

</Prose>
</div>

<div class="space-y-4">
<Prose>
#### 2. Configure path aliases

Add to your `tsconfig.json`:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```

</Prose>
</div>
